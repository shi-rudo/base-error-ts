# Project conventions

## Typography

Em dashes (`—`) are forbidden in prose. This applies everywhere the project
writes sentences: Markdown docs, JSDoc, code comments, commit messages, and PR
descriptions.

Restructure the sentence instead of reaching for an em dash:

- Split it into two sentences with a period.
- Use a colon when what follows explains or expands the first clause.
- Use parentheses for a genuine aside.
- Use a comma where the break is light.

Do not substitute an em dash with a spaced hyphen (` - `) or an en dash (`–`)
either; choose real punctuation that fits the sentence.
